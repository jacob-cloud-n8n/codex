# 我的第二大腦 — CLAUDE.md

## 關於我

- 這個 vault 是我的教學與班級工具專案第二大腦。
- 目前專案是 `2026 Codex`，用來建立 Codex、GitHub、Obsidian、Firebase 與 YouTube 工作流。

## 語言偏好

- 所有回應請使用繁體中文。
- 筆記內容也使用繁體中文。
- 回答要清楚、簡潔、可執行。

## 筆記庫結構（三層）

| 資料夾 | 用途 | 存什麼 |
|---|---|---|
| `Clippings/` | 輸入 | 網路剪藏、文章、影片筆記、外部資料 |
| `知識庫/` | 消化 | AI 整理後的結構化知識、索引、操作紀錄 |
| `創作庫/` | 輸出 | 原創內容、教案、教材、企劃、可交付文件 |
| `每日筆記/` | 時間管理 | 每日紀錄、週計畫、知識重整週報 |
| `Templates/` | 模板 | 各種筆記固定格式 |

## 工作規則

### 新增筆記時

- 一律加上 frontmatter：`title`、`date`、`type`、`tags`。
- 根據內容判斷存到正確資料夾：
  - 別人的文章或原始資料 → `Clippings/`
  - AI 整理後的知識 → `知識庫/`
  - 原創作品或規格 → `創作庫/`
  - 每日/每週紀錄 → `每日筆記/`

### Clippings 處理規則

- `Clippings/` 是原始資料區，原文不要直接改寫。
- 消化資料時，另存到 `知識庫/`，並保留來源連結。

### 知識庫維護規則

- `知識庫/index.md` 是索引。
- `知識庫/log.md` 是操作紀錄。
- 每次新增知識庫頁面，都要更新 `index.md`。
- 每次做知識重整，都要在 `log.md` 新增紀錄。

### 使用者說「幫我新增到筆記」時

- 代表存到這個 Obsidian vault。
- 先判斷內容屬於輸入、消化、輸出或時間紀錄，再放到對應資料夾。

## 與 Codex 的分工

- `CLAUDE.md`：第二大腦筆記庫規則，方便 Claude Code 或其他工具讀取。
