# 歡迎來到專案第二大腦

這個 vault 會同時支援專案開發、GitHub 協作與 Obsidian 知識管理。

## 你的筆記庫結構

| 資料夾 | 用途 | 你要做的事 |
|---|---|---|
| `Clippings/` | 輸入 | 收集外部文章、想法、參考資料 |
| `知識庫/` | 消化 | 放整理後的概念、決策與專案知識 |
| `創作庫/` | 輸出 | 放規格、企劃、文章、可交付文件 |
| `每日筆記/` | 時間管理 | 記錄每日進度、週計畫與回顧 |
| `Templates/` | 模板 | 建立一致格式的筆記 |

## 常用說法

| 你說的話 | Codex 會做的事 |
|---|---|
| 「新增到筆記」 | 判斷內容並存到合適資料夾 |
| 「消化資料」 | 從 `Clippings/` 整理到 `知識庫/` |
| 「跑一次知識重整」 | 盤點近期筆記、更新索引與紀錄 |
| 「整理專案狀態」 | 更新任務看板、開發日誌與 README |

## 建議使用方式

平時把未整理資料放進 `Clippings/`。當你需要推進專案、整理研究或建立規格時，再請 Codex 把資料消化成 `知識庫/` 或 `創作庫/` 的文件。
