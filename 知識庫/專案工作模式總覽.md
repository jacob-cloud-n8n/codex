# 專案工作模式總覽

## 核心結構

本專案使用「工作桌 + 三個家」：

| 名稱 | 位置 | 用途 |
|---|---|---|
| 工作桌 | Google Drive 專案資料夾 | 實際開發與檔案管理 |
| GitHub | `jacob-cloud-n8n/codex` | 版本控制與同步 |
| Obsidian | 目前專案資料夾 | 筆記、駕駛艙、知識庫 |
| Firebase | `codex-jacob` | 即時互動資料 |

## 三個流程

| 觸發詞 | Skill | 用途 |
|---|---|---|
| 開工 | `startup-sync` | 讀規則、讀駕駛艙、檢查 Git/Firebase、建議下一步 |
| 收工 | `shutdown-sync` | 更新駕駛艙、提交相關變更、推送 GitHub |
| 新專案初始化 | `project-init-sync` | 盤點並補齊新/既有專案工作模式 |

## 固定規則

- `AGENTS.md` 放固定規則。
- `docs/專案工作流程.md` 放進度與踩坑。
- `知識庫/` 放整理後的可重用知識。
- `每日筆記/` 放每天或每週的工作紀錄。
