## 踩坑筆記

- Firebase standalone CLI 第一次啟動需要寫入 `~/.cache/firebase`，在 Codex 沙箱中需要授權。
- `firestore_add_document` 的 `document` 參數要傳 Firestore document 物件，不要包成 JSON 字串。
- Codex 改完 MCP 設定後，要完整重啟 Desktop 才會載入工具。
- 終端機與 Codex 的 PATH 可能不同；必要時用完整路徑 `/Users/jacob/.local/bin/firebase`。
- Gemini CLI 第一次執行會建立 `~/.gemini`；若在沙箱中報權限錯誤，先建立該資料夾再驗證。
- `garrytan/gstack` 安裝流程會寫入 `~/.codex/skills`、`~/.gstack`，建置 Playwright/Bun 工具，並包含 telemetry、瀏覽器 cookie 匯入、gbrain/記憶同步與私有 repo artifact sync；列入觀察名單，非必要不安裝。若未來確實需要，只審查並安裝單一必要功能，不安裝完整版。

## 開工流程

說「開工」時：

1. 讀 `AGENTS.md`
2. 讀本檔
3. 檢查 Git 狀態
4. 檢查 Firebase MCP 狀態
5. 回報下一步

## 收工流程

說「收工」時：

1. 盤點本次工作
2. 更新本檔
3. 檢查 Git diff
4. 只提交本次相關檔案
5. push 到 GitHub
